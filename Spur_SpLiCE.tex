\begin{abstract}
Self-supervised visual encoders can exploit spurious correlations even when class labels are absent from their pretraining objective. This paper asks whether sparse, language-aligned concepts from a frozen vision–language model can expose spurious structure and guide a separate SSL encoder during training. Our initial pipeline decomposes frozen CLIP embeddings with SpLiCE, ranks concepts associated with a known spurious attribute conditional on the target label and injects the resulting signal into SimCLR through routed augmentation or class-conditional correlation regularization. On Waterbirds, neither interface yields a reliable worst-group improvement over matched SimCLR. The strongest broad-sweep configuration improves mean worst-group validation accuracy by only 0.45 points, while Top-5 semantic routing underperforms a matched shuffled-routing control on all three paired seeds. These controls isolate the bottleneck in how the concept signal is used. We introduce a synthesis–distillation formulation as the main hypothesis. For each image, we edit only the selected spurious-concept coordinates of its sparse SpLiCE code, reconstruct a residual-preserving counterfactual CLIP embedding and distill this synthetic target into the SSL encoder through a dedicated projection head and a stop-gradient cosine loss. Unlike minibatch positive mining, the proposed mechanism does not require a suitable same-class, cross-background example to occur in the current batch. The result is negative: routing and regularization fail, but the failure is diagnostic. We therefore pivot from scalar routing to concept-edited representation targets.
\end{abstract}

\section{Introduction}
A ResNet18 trained with SimCLR on Waterbirds reaches 48\% worst-group accuracy, but this masks a 30-point gap between majority and minority groups. Does an interpretable frozen vision–language model expose enough structure to guide a separate SSL encoder? We use SpLiCE to decompose CLIP embeddings into sparse concept vectors and rank concepts associated with the spurious attribute Our approach uses Sparse Linear Concept Embeddings (SpLiCE) \citep{bhalla2024splice} to approximate frozen CLIP image embeddings as sparse nonnegative combinations of text-concept embeddings. We automatically rank concepts according to their association with the known spurious attribute after conditioning on the target label. The initial interventions use these coordinates to (i) route selected images to a stronger SimCLR augmentation branch or (ii) penalize class-conditional correlations between SSL features and selected concept coordinates.

The question is whether SpLiCE per-image concept geometry can induce an invariance in an independent encoder. We therefore compare semantic routing against matched shuffled controls and evaluate worst-group (WG) accuracy with paired seeds. The evidence is negative for the two implemented interfaces: neither routed augmentation nor coordinate-wise correlation regularization reliably improves WG accuracy and semantic routing performs worse than shuffled routing across all three paired seeds. The failure is controlled and it points to a specific bottleneck: the scalar routing mechanism itself. A scalar routing decision specifies \emph{where} to apply a generic transform, but not \emph{which unneccessary component}  changes or \emph{which representation} the encoder should approach.

This diagnosis suggests a different approach: \emph{synthesize a concept-edited target, then distill it}. Instead of searching a minibatch for an observed counterfactual, we edit the selected nuisance coordinates of each image's sparse code and use the SpLiCE dictionary to synthesize a residual-preserving CLIP target. A separate head then distills this target into the SSL representation while the standard SimCLR head retains the instance-discrimination objective. The construction specifies the representation change directly and handles rare groups: it does not require a batch to contain a same-class, cross-background partner.

Our current contributions are:
\begin{itemize}
    \item an end-to-end pipeline that connects frozen SpLiCE concept discovery to the training of a separate SimCLR encoder;
    \item a target-conditioned concept-ranking procedure and a vector-valued correlation regularizer operating on per-image SpLiCE coordinates;
    \item a multi-seed matched control showing that Top-K semantic routing
    underperforms shuffled routing, isolating a failure of the current
    concept-to-augmentation interface;
    \item an empirical diagnosis showing that plausible concept detection is
    insufficient for robust improvement under the two simple interventions
    studied here;
    \item a falsifiable synthesis--distillation formulation that edits only
    nuisance-designated SpLiCE coordinates, reconstructs a counterfactual CLIP
    target and transfers it through a dedicated stopped-gradient distillation
    head, together with controls that isolate editing, coordinate selection,
    and donor selection.
\end{itemize}

\section{Related Work}
\paragraph{Sparse concept decompositions.}
SpLiCE decomposes dense CLIP embeddings into sparse, nonnegative combinations of language-aligned concept embeddings and has been proposed for interpretation, concept bottlenecks, bias discovery and representation editing \citep{bhalla2024splice}. We instead freeze SpLiCE and use it to audit and control a separate ResNet encoder trained from scratch.

\paragraph{Robust self-supervised learning.}
Prior work has shown that SSL representations can encode and exploit spurious features and has proposed feature-space augmentation or training-dynamics-based interventions \citep{hamidieh2024views,zhu2023lassl}. Our intervention signal comes from a frozen vision–language model: we extract interpretable coordinates that encode the spurious dimension

\paragraph{Concept discovery for spurious mitigation.}
Concept discovery has also been used to rebalance supervised classifier training. CoBalT, for example, learns dataset-specific object-centric latent concepts and uses them for concept-balanced sampling \citep{arefin2024cobalt}. Prior work uses concepts to rebalance supervised classifier training. We ask whether they can instead shape SSL pretraining directly.

\paragraph{Post-hoc concept auditing and visual projection.}
Bias Leaves a Gradient Trail ranks concepts from an activation decomposition by their interaction with gradients on model errors, then suppresses the highest-ranked directions in a frozen classifier \citep{vitry2026gradient}. It demonstrates that concept decompositions can expose decision-relevant bias without spurious-attribute labels or model retraining. VisualDistiller instead projects CLIP image embeddings away from a subspace spanned by retrieved or generated background images and trains a downstream linear predictor on the resulting visual representation \citep{dai2024visualdistiller}. These methods operate on a deployed representation or decision model. We likewise edit a frozen CLIP representation, but use the edited embedding as a teacher target for training a separate SSL encoder.

\paragraph{Generative variants and cross-variant SSL.}
Generative Randomization and Cross-Variant SSL constructs multiple pixel-space contexts for the same segmented foreground and treats the resulting variants as contrastive positives \citep{yadav2026crossvariant}. Its reported $92.5\%$ Waterbirds WG accuracy makes it an important direct comparison for the invariance sought here. Our proposed formulation avoids foreground segmentation and diffusion generation: it synthesizes the counterfactual in the CLIP--SpLiCE representation space and transfers it by distillation. This substitution is cheaper and removes dependence on an observed batch partner, but its faithfulness depends on the quality and local linearity of the sparse concept edit.

\section{Problem Setting}
Let each image $x_i$ have a target label $y_i$ and a spurious attribute $a_i$. The SSL encoder $f_\theta$ is trained with the SimCLR NT-Xent objective on two augmented views of each image \citep{chen2020simclr}. Current concept-discovery and conditional-regularization variants use metadata during development, even though target and spurious labels are not prediction targets of the SSL encoder. Thus we describe this method as \emph{metadata-assisted SSL}.

The main evaluation criterion is worst-group accuracy over groups defined by $(y,a)$. Hyperparameters are selected on validation data and the test split is reserved for a final preselected configuration. A successful intervention should improve WG accuracy relative to a matched SimCLR baseline. A spurious-attribute probe is used as a diagnostic of linearly accessible nuisance information, but high probe accuracy alone is not treated as proof that the downstream classifier causally relies on the nuisance attribute.

\section{Method}
\subsection{Frozen SpLiCE decomposition}
A frozen CLIP encoder maps an image to a dense embedding. SpLiCE approximates this embedding using a sparse nonnegative vector over a vocabulary of text concepts. We use \texttt{open\_clip:ViT-B-32} with \texttt{laion2b\_s34b\_b79k} weights, the top 10,000 LAION vocabulary entries and an $\ell_1$ penalty of $0.25$. The decomposition is computed once for the training split and cached.

For image $x_i$, let $c_i\in\mathbb{R}_{\geq0}^{K}$ denote the selected concept coordinates. The broad sweep uses ten concepts:
\begin{center}
\small\texttt{bamboo, forests, whale, rainforest, raven, forest, seascape, lake, snorkeling, pond}.
\end{center}
The routing-control series uses an automatically deduplicated and direction-consistent top five:
\begin{center}
\small\texttt{bamboo, forests, whale, rainforest, raven}.
\end{center}

\subsection{Automatic nuisance-concept ranking}
For concept $k$, target $y$ and spurious value $a$, let $\mu_{a,y,k}$ be the mean SpLiCE weight in that group. For binary attributes, we compute a target-conditioned spurious effect
\begin{equation}
S_k = \frac{1}{|\mathcal{Y}|}\sum_{y\in\mathcal{Y}}\left|\mu_{1,y,k}-\mu_{0,y,k}\right|,
\end{equation}
a spurious-conditioned target effect
\begin{equation}
T_k = \frac{1}{|\mathcal{A}|}\sum_{a\in\mathcal{A}}\left|\mu_{a,1,k}-\mu_{a,0,k}\right|,
\end{equation}
and an instability term that penalizes disagreement in the magnitude of the spurious effect across target classes. With unit coefficients, the ranking score is
\begin{equation}
\mathrm{Score}_k = S_k-T_k-I_k.
\end{equation}
The revised control series additionally requires the signed effect $d_{k,y}=\mu_{1,y,k}-\mu_{0,y,k}$ to have the same direction for every target class and removes simple lexical duplicates before selecting the top $K$ concepts.

\subsection{Concept-routed augmentation}
For routing, the selected vector is reduced to
\begin{equation}
r(x_i)=\max_k c_{ik}.
\end{equation}
A threshold is set by a training-set quantile $q$. Images above the threshold produce one standard and one strong view,
\begin{equation}
\left[T_{\mathrm{std}}(x),T_{\mathrm{strong}}(x)\right],
\end{equation}
whereas images below the threshold produce two standard views. At $q=0.75$, semantic, shuffled and matched-random policies each route exactly 1,199 of 4,795 training images of the waterbirds dataset (25.01\%). The augment-all control routes every image.

\subsection{Conditional correlation regularization}
The SSL encoder produces $h_i=f_\theta(x_i)\in\mathbb{R}^{d}$. Because target and spurious attributes are correlated in the biased training set, embeddings and concept vectors are centered separately within each target class:
\begin{equation}
\widetilde h_i=h_i-\frac{1}{|B_{y_i}|}\sum_{j\in B_{y_i}}h_j,
\qquad
\widetilde c_i=c_i-\frac{1}{|B_{y_i}|}\sum_{j\in B_{y_i}}c_j.
\end{equation}
The feature--concept correlation matrix is
\begin{equation}
R_{jk}=\frac{\sum_i\widetilde h_{ij}\widetilde c_{ik}}
{\sqrt{\sum_i\widetilde h_{ij}^{2}}\sqrt{\sum_i\widetilde c_{ik}^{2}}+\epsilon},
\end{equation}
and the regularized objective is
\begin{equation}
\mathcal{L}=\mathcal{L}_{\mathrm{SimCLR}}+
\lambda\frac{1}{dK}\sum_{j=1}^{d}\sum_{k=1}^{K}R_{jk}^{2}.
\end{equation}
This penalty discourages linear accessibility of each selected concept coordinate within a fixed target class, but does not guarantee removal of nonlinear or distributed nuisance information.

\subsection{Concept synthesis and distillation}
\label{sec:synthesis_distillation}
The routing intervention compresses $c_i$ to a scalar and assigns a generic image transform. We instead construct an explicit representation target in which only the selected nuisance contribution has changed. This formulation is the next method; the results in Section~5 evaluate the preceding interventions and do not yet include the synthesis--distillation runs.

Let $z_i\in\mathbb{R}^{d_c}$ be the normalized frozen CLIP embedding and let $D\in\mathbb{R}^{K\times d_c}$ contain the normalized SpLiCE dictionary directions as rows. SpLiCE does not decompose $z_i$ directly: it solves the nonnegative sparse problem in the mean-centered embedding space, so that
$\operatorname{norm}(z_i-\mu)\approx c_iD$, where $\mu$ is the dataset-level CLIP image mean supplied with the frozen decomposition. For the selected nuisance coordinates $\mathcal{S}$, an intervention produces $c'_{i,\mathcal{S}}$ while leaving $c'_{i,\overline{\mathcal{S}}}=c_{i,\overline{\mathcal{S}}}$. We construct the edited target with the local residual-preserving update
\begin{equation}
\widetilde z_i =
\operatorname{norm}\!\left(
z_i+\alpha\left(c'_{i,\mathcal{S}}-c_{i,\mathcal{S}}\right)D_{\mathcal{S}}
\right),
\label{eq:synthetic_target}
\end{equation}
where $\alpha$ controls intervention strength. First, at $\alpha=1$ Equation~\ref{eq:synthetic_target} is algebraically identical to explicitly retaining the unexplained part of the embedding and re-adding the edited code, $\operatorname{norm}\!\left((z_i-c_iD)+c'_iD\right)$, so no CLIP information outside the sparse support is discarded ---  whereas reconstructing the target solely from $c'_iD$, which would delete everything the sparse code fails to explain. Second, because the update depends only on the \emph{difference} of two codes expressed in the same centered basis, the offset $\mu$ cancels exactly; the edit is therefore well defined despite the decomposition being centered while $z_i$ is not. The magnitude of the applied displacement is nonetheless expressed in the units of the centered space, which is one reason $\alpha$ is swept rather than fixed at one.

We consider three semantic edits. \emph{Zero-out} sets
$c'_{i,\mathcal{S}}=0$. \emph{Class-conditional neutralization} replaces each
selected coordinate by its training-set median within target class $y_i$,
\begin{equation}
c'_{i,\mathcal{S}}
=
\operatorname{median}_{j:\,y_j=y_i}c_{j,\mathcal{S}}.
\label{eq:class_neutralization}
\end{equation}
Finally, an oracle \emph{core-matched swap} selects a same-class donor from the opposite spurious group whose residual representation is maximally similar to that of the anchor and sets
$c'_{i,\mathcal{S}}=c_{j^\star,\mathcal{S}}$. The oracle uses $a_i$ only to determine whether a correctly specified cross-background edit can help. Random-coordinate, shuffled-donor and same-class-random-donor variants form the corresponding attribution controls.

The trainable encoder $f_\theta$ is shared by two projection heads. The standard head $g_{\mathrm{ssl}}$ is optimized with NT-Xent, while a separate head $g_{\mathrm{clip}}$ predicts the synthetic CLIP target. For two augmented views $t_1(x_i)$ and $t_2(x_i)$, both views receive the same image-level target $\widetilde z_i$. The distillation term is
\begin{equation}
\mathcal{L}_{\mathrm{distill}}
=
\frac{1}{2|B|}
\sum_{i\in B}\sum_{v=1}^{2}
\left[
1-
\cos\!\left(
g_{\mathrm{clip}}\!\left(f_\theta(t_v(x_i))\right),
\operatorname{sg}[\widetilde z_i]
\right)
\right],
\label{eq:distillation}
\end{equation}
and the complete objective is
\begin{equation}
\mathcal{L}
=
\mathcal{L}_{\mathrm{NT\text{-}Xent}}
\!\left(
g_{\mathrm{ssl}}(f_\theta(t_1(x_i))),
g_{\mathrm{ssl}}(f_\theta(t_2(x_i)))
\right)
+\lambda\mathcal{L}_{\mathrm{distill}}.
\label{eq:synthesis_objective}
\end{equation}
Here $\operatorname{sg}$ stops gradients through frozen CLIP, SpLiCE and the edit. Synthetic targets are generated once and cached; only $f_\theta$, $g_{\mathrm{ssl}}$ and $g_{\mathrm{clip}}$ are updated. This design separates instance discrimination from teacher-space alignment and avoids a practical weakness of minibatch mining: with rare conflict groups, a batch of 256 need not contain a suitable same-class, cross-background positive for every anchor.

\section{Experimental Setup}
\subsection{Dataset and evaluation}
We evaluate on Waterbirds. A ResNet18-large encoder is trained with SimCLR; the completed interventions reported in Section~5 use 1,000 epochs, while the synthesis--distillation stage uses the shortened 500-epoch protocol described in Section~\ref{sec:staged_evaluation}. A linear target probe is trained on the group-balanced \texttt{ds\_train} split and evaluated on validation data every 25 SSL epochs. Run summaries use the mean of the final ten probe evaluations. Comparable aggregates use seeds 1, 3 and 4. Because run length changes the absolute accuracy level, every comparison is made only within a fixed epoch budget against a baseline trained under that same budget.

\subsection{Staged evaluation of synthesis--distillation}
\label{sec:staged_evaluation}
We separate validation of the mechanism from expensive SSL training. First, a frozen-CLIP linear-probe sanity check evaluates whether zeroing the selected SpLiCE coordinates changes the expected group behavior; this reproduces the zero-out ablation from the SpLiCE appendix using our concept list and is not treated as a contribution. Second, a one-seed pilot compares SimCLR, distillation to the unedited CLIP embedding and distillation to the edited target. We expand to the full control battery in Appendix~\ref{app:control_battery} and five paired seeds only if the edited target separates from both baseline and the unedited-teacher control. This ordering prevents a long ResNet sweep from obscuring a failure of the frozen teacher edit itself.

Two hyperparameters govern the proposed objective. The distillation weight is swept over $\lambda\in\{0.05,0.1,0.25,1.0\}$ and the intervention strength over $\alpha\in\{0.5,1.0,1.5\}$, with $\lambda=0.1$ and $\alpha=1.0$ as the pilot defaults. Rather than tuning $\lambda$ by its nominal value alone, we monitor the realized ratio $\mathcal{L}_{\mathrm{distill}}/\mathcal{L}_{\mathrm{NT\text{-}Xent}}$, because the earlier correlation regularizer was found to be numerically inactive at small weights and destructive at large ones. All synthesis--distillation runs use a shortened 500-epoch schedule with learning-rate milestones rescaled to $70\%$, $80\%$ and $90\%$ of the budget. This protocol is internally matched across every configuration in the control battery, but it is not directly comparable with the 1,000-epoch legacy tables reported in Section~5.

\subsection{Strong-augmentation components}
The all-components configuration also fails to outperform the baseline. Single-component crop, color-jitter, grayscale and Gaussian-blur results currently have only one comparable seed each and therefore cannot support a stability claim. Spurious-attribute probe accuracy remains high across these runs (approximately 90.40--92.49\% average validation accuracy for seed 1), so the current augmentation policy does not provide evidence that background information has ceased to be linearly accessible.

\section{Discussion}
\paragraph{What the negative control establishes.}
Detecting spurious concepts is not the same as using them to induce invariance. SpLiCE produces human-readable coordinates associated with the Waterbirds background, but a large scalar concept score is not a prescription for how an individual image should be transformed. Strong generic transforms may preserve useful information, damage the foreground or make instance discrimination unnecessarily difficult. The all-negative semantic-minus-shuffled paired differences provide direct evidence against the specific routing interface: for a fixed augmentation budget, the semantic assignment is not better than destroying that assignment. The experiment does not, however, test whether pairwise relations between the concept vectors are useful.

\paragraph{Why the next hypothesis is synthesis--distillation.}
Neither implemented intervention specifies a counterfactual representation for a particular image. Routing reduces $c_i$ to $\max_k c_{ik}$ and changes generic augmentation strength, while correlation regularization suppresses marginal linear dependence without identifying the representation that should replace it. Section~\ref{sec:synthesis_distillation} instead changes only the selected sparse coordinates and preserves the remaining CLIP residual. The resulting target states both the nuisance component to modify and the representation toward which the encoder should move. A dedicated $g_{\mathrm{clip}}$ head prevents this teacher-space constraint from replacing the SimCLR projection geometry.

This mechanism also resolves a sampling problem in the previously considered positive-mining formulation. Waterbirds contains rare conflict groups and a minibatch of 256 may contain few or no same-class examples with sufficiently different backgrounds. Mining therefore conflates the quality of the concept signal with the availability of an appropriate donor in the current batch. Synthesis produces one counterfactual target per image before training; donor-based variants may search the full training split, while neutralization requires no donor at all.

\paragraph{Attribution protocol for the proposed method.}
An improvement over SimCLR alone would not establish that sparse editing is useful: ordinary CLIP distillation may already be a stronger teacher. The first necessary comparison is therefore the unedited CLIP target. This control is two-sided and informative in either direction. If the unedited teacher already captures most of the gain, the sparse edit is redundant; if instead the unedited teacher \emph{degrades} worst-group accuracy, that is direct evidence that the frozen CLIP representation transfers its own background bias into the student and the edited target must be judged against that degraded reference. Random-coordinate neutralization tests whether the selected nuisance coordinates matter, while shuffled and same-class-random donors test whether donor semantics matter. Zero-out tests the simplest removal hypothesis and the metadata-assisted core-matched cross-background swap estimates the upper bound when the intervention is correctly specified. Only an edited target that improves WG accuracy over both SimCLR and unedited CLIP distillation, survives these controls across paired seeds and does not sacrifice average accuracy would support the synthesis claim.

\section{Limitations}
The present study is restricted to Waterbirds and three seeds are insufficient for strong statistical claims about the completed interventions. Concept ranking and several synthesis controls use target or spurious metadata, so the current study is metadata-assisted SSL. Selected concepts such as \texttt{whale} and \texttt{raven} may be imperfect proxies for background structure. More fundamentally, Equation~\ref{eq:synthetic_target} assumes that a local linear edit along SpLiCE dictionary directions produces a meaningful CLIP counterfactual. The frozen teacher may itself encode the same bias, coordinate changes can interact outside the sparse support and the edited embedding need not correspond to a realizable image. The separate distillation head limits interference but does not guarantee that invariance transfers to the encoder features used by the downstream probe. The synthesis stage also uses a shortened 500-epoch budget, which lowers absolute accuracy relative to the 1,000-epoch tables and may understate methods whose benefit appears late in training; it is applied identically to every configuration in the battery, so within-stage comparisons remain paired, but cross-stage numbers must not be read as a single benchmark. Finally, synthesis--distillation has not yet been evaluated on the held-out test set; it remains a falsifiable next method, not a positive empirical result of the current report.

\section{Conclusion}
We presented a reproducible framework for connecting frozen SpLiCE concepts to the training and evaluation of a separate self-supervised encoder. On Waterbirds, neither concept-routed augmentation nor target-conditioned correlation regularization yields a reliable WG improvement over matched SimCLR. The completed three-seed control is more diagnostic: semantic Top-5 routing underperforms shuffled routing on every paired seed. This negative result rules out the current scalar routing mechanism, but not the broader hypothesis that concept geometry can guide robust representation learning. The project therefore moves to a more explicit interface: edit nuisance-designated sparse coordinates, synthesize a residual-preserving CLIP target and distill it through a separate stopped-gradient head while retaining NT-Xent. The next experiment asks whether edited targets outperform unedited CLIP distillation and non-semantic controls.

\section{Frozen-Teacher Sanity Check}
Before SSL training, we apply the SpLiCE concept-bottleneck classifier to the unedited and zeroed sparse representations using the concept list selected in this study. This follows the zero-out ablation in the SpLiCE appendix and answers a narrow implementation question: does our code remove the intended coordinates and measurably alter the frozen-model prediction? We report this check separately and do not count it as a contribution, a new intervention, or evidence that distillation will improve WG accuracy.

\section{Reproduction Notes}
The main entry point is \texttt{spur\_splice.py}. Supported SpLiCE modes are \texttt{none}, \texttt{augment}, \texttt{corr\_reg}, \texttt{augment\_corr\_reg} and \texttt{synthesis\_distill}. The latter precomputes residual-preserving targets, attaches the dedicated \texttt{clip\_distillation\_head} and optimizes the stopped-gradient cosine term jointly with SimCLR. Its interventions are \texttt{original}, \texttt{class\_neutralize}, \texttt{random\_coords}, \texttt{shuffled\_donor}, \texttt{same\_class\_random\_donor}, \texttt{zero\_out} and \texttt{core\_matched\_swap}. Synthesized teacher targets are cached per dataset, concept list, intervention and strength, so the control battery pays the frozen-CLIP cost once per configuration.

Two launchers expose exactly the battery in Table~\ref{tab:synthesis_controls}. On Windows, \texttt{scripts/Run-HomeExperiments.ps1 -Family synthesis -Tasks 0,1,2 -Seeds 4} runs the pilot; on Slurm, \texttt{scripts/waterbirds\_SpLiCE\_synthesis\_array.sbatch} maps array indices $0$--$7$ to the same tasks and serializes automatic concept discovery with a file lock so that concurrent array tasks share one concept list. Both default to the 500-epoch protocol and name W\&B runs as \texttt{\{Dataset\}\_S\{seed\}\_\{Task\}\_w\{$\lambda$\}a\{$\alpha$\}\_e\{epochs\}}, which keeps configurations distinguishable when $\lambda$ or $\alpha$ is swept. Final test evaluation still requires the explicit \texttt{--final\_test} flag.
